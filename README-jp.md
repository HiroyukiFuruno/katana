# Katana

> Rust と egui で構築された、macOS 向けの高速でキーボード中心の Markdown エディタ。

Katana は Markdown ドキュメントの執筆とプレビューに特化したネイティブデスクトップアプリケーションです。
編集内容をリアルタイムにプレビューしながら執筆でき、図のコード化（Mermaid・PlantUML・Draw.io）にも対応しています。
Electron や Node.js は不使用 — 単一のネイティブバイナリで、軽量かつ高速に動作します。

---

## 主な機能

- **ライブ分割ビュープレビュー** — 左でコード編集、右で HTML プレビューをスクロール連動表示
- **図のレンダリング** — Mermaid・PlantUML・Draw.io のフェンスコードブロックをネイティブ描画
- **GitHub Flavored Markdown** — テーブル・取り消し線・タスクリスト・脚注・自動リンクに対応
- **ワークスペース対応** — フォルダを開いてファイルツリーから直接ナビゲーション
- **タブバー** — VSCode スタイルのタブで複数ドキュメントを同時に開ける
- **国際化（i18n）** — UI 文字列の完全ローカライズ（英語・日本語を同梱）
- **プラグインシステム** — 将来の図バックエンドを追加できる拡張ポイント設計
- **AI 連携フック** — 将来の AI 支援ワークフロー向けプロバイダーレジストリを搭載
- **macOS ネイティブ** — ネイティブメニューバー、CJK フォント対応（AquaKana / ヒラギノ角ゴ）

## 開発環境の構築

### 前提条件

- macOS 13 Ventura 以降
- [Homebrew](https://brew.sh)（セットアップスクリプトが未インストールの場合に自動インストールします）

### ワンコマンドセットアップ

プロジェクトルートから以下のスクリプトを実行してください。
必要なツールを確認しながら対話形式でインストールします。

```sh
./scripts/setup.sh
```

インストールされるツール：

| ツール | 用途 |
| --- | --- |
| **Homebrew** | macOS パッケージマネージャー |
| **git**（最新版） | バージョン管理 |
| **rustup** | Rust ツールチェーン管理 |
| **Rust stable** + clippy / rustfmt / llvm-tools | コンパイラとリンター |
| **cargo-llvm-cov** | 行カバレッジゲート（CI で 100% 強制） |
| **cargo-watch** | ファイル変更時の自動リビルド |
| **cargo-outdated** | 陳腐化した依存クレートの検出 |
| **cargo-bloat** | バイナリサイズ分析 |
| **tokei** | ソースコード行数カウント |
| **lefthook** | Git フックランナー（pre-commit + pre-push） |

### よく使う `make` コマンド

```sh
make run          # ビルドして Katana を起動
make test         # テストスイート全体を実行
make ci           # CI と同等のフルチェック: fmt + clippy + テスト
make watch-run    # ファイル変更監視 & 自動再起動
make doc-open     # API ドキュメントをビルドしてブラウザで開く
```

すべてのコマンドは `make help` で確認できます。

### プロジェクト構成

```text
katana/
├── crates/
│   ├── katana-core/      # Markdown パイプライン、ドキュメントモデル、ワークスペース、AI レジストリ
│   ├── katana-ui/        # egui アプリシェル、プレビューペイン、i18n、スナップショット
│   └── katana-platform/  # ファイルシステム抽象化、設定の永続化
├── scripts/
│   └── setup.sh          # 開発環境インストーラー
├── docs/                 # アーキテクチャ決定記録、コーディングルール
├── fixtures/             # テスト用フィクスチャ（サンプル Markdown ファイル）
└── Makefile              # 開発者向けタスクランナー
```

### 品質ゲート

| ゲート | 実行タイミング | チェック内容 |
| --- | --- | --- |
| pre-commit | `git commit` のたび | `cargo fmt --check`、`cargo clippy -D warnings` |
| pre-push | `git push` のたび | テストスイート全体 + `cargo llvm-cov`（行カバレッジ 100%） |
| CI | PR / `master` へのプッシュ | fmt · clippy · テスト · カバレッジ · CodeQL セキュリティスキャン |

## OSS 参加（コントリビューション）

Katana は現在、初期の活発な開発フェーズにあります。あらゆる形での貢献を歓迎します。

### 貢献の方法

- **バグ報告** — 再現手順を添えて [Issue](../../issues) を開いてください
- **機能リクエスト** — コードを書く前に Issue で議論しましょう
- **コード** — リポジトリをフォークし、ブランチで実装してプルリクエストを作成してください
- **ドキュメント** — 誤記・説明の改善・翻訳もいつでも歓迎です

### プルリクエストのガイドライン

1. リポジトリをフォークし、`master` からブランチを作成する
2. PR を開く前に `make ci` が完全に通ることを確認する
3. 新機能にはテストを追加する（カバレッジゲート: 行 100%）
4. コミットは小さくまとめ、明確なメッセージを書く
5. アプローチについてフィードバックがほしい場合はドラフト PR を早めに開く

### コードスタイル

lefthook によるフォーマットとリンティングが自動で強制されます。
主なルールは以下の通りです。

- **rustfmt** — `max_width = 100`、Unix 改行（`cargo fmt` で強制）
- **clippy** — 全警告をエラー扱い（`-D warnings`）、関数本体 ≤ 30 行、認知的複雑度 ≤ 10
- **i18n** — ユーザーに表示される文字列はすべて `i18n!()` マクロ経由で — UI テキストのハードコード禁止
- 詳細は `docs/coding-rules.md` を参照してください
