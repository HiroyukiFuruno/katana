# Katana 統合試験シナリオ (egui_kittest)

本ドキュメントでは、将来フェーズ（egui 0.30 アップグレード以降）で実装する `egui_kittest` ベースの統合試験（UIテスト）のシナリオと実装方針を整理します。

## 概要

`egui_kittest` の特徴（AccessKit ノード検索、`wgpu` によるスナップショット生成）を活用し、「ユーザー視点のユースケース」と「レイアウト/描画結果」を担保します。テストは `katana-ui/tests/e2e/` ディレクトリ配下に定義し、ヘッドレスな EG 状態でのみ動作させます。

## シナリオ一覧

### a. ワークスペースとドキュメント開閉

**目的**: 開発ディレクトリを正しくマウントし、ファイルがタブとして開けることを保証する。

* **ステップ 1**: テンポラリディレクトリにファイル構成（`test1.md`, `dir/test2.md`）を生成する。
* **ステップ 2**: アプリケーション Harness を起動する。
* **ステップ 3**: `OpenWorkspace` アクションを発行する（UI ダイアログモック経由）。
* **ステップ 4**: ワークスペースのツリービューに "test1.md" と "dir" が表示されるかアサートする。
* **ステップ 5**: "test1.md" のラベル（ノード）を取得しクリックする。
* **ステップ 6**: アプリのタブバーに "test1.md" が表示されるかアサートする。

### b. テキスト編集とプレビュー同期

**目的**: 中央のテキストエディタの変更が、別ペインのマークダウン描画へ即時反映されることを保証する。

* **ステップ 1**: ファイルを開き、編集領域（`egui::TextEdit`）を検索する。
* **ステップ 2**: テキストボックスに `# Hello Katana` という文字列を入力する。
* **ステップ 3**: プレビューペインを検索し、「Hello Katana」というテキストを持つ見出しレイヤーが生成されているかをアサートする。
* **ステップ 4**: タブバーに Dirty フラグ（未保存: `*`）が表示されることを確認する。
* **ステップ 5**: ネイティブメニューのファイル保存（SaveDocument）アクションをシミュレートし、Dirty フラグが消えるかアサートする。

### c. 表示モード切り替えとタブナビゲーション

**目的**: "ViewMode" （Preview / Code / Split）のトグルとペイン制御の正当性を保証する。

* **ステップ 1**: "Preview" （プレビューのみ）が選択されている時、エディタ部分が非表示であることをアサートする。
* **ステップ 2**: "Split" ボタンをクリックする。
* **ステップ 3**: エディタペインとプレビューペインの双方が存在し、水平に並んでいる（横幅を分け合っている）ことを寸法としてアサートする。
* **ステップ 4**: "Code" ボタンをクリックし、プレビューペインが非表示になることをアサートする。
* **ステップ 5**: 複数のタブを開き、"◀" と "▶" ボタンをクリックして、アクティブタブがサイクリックに切り替わるかをアサートする。

### d. スナップショットテスト一覧 (Visual Regression)

ピクセルレベルでの退行バグを防ぐため、以下のスナップショットテスト（`Harness::snapshot()`）を実行します。

| No. | ターゲット画像名 | シチュエーション | 確認ポイント |
| --- | --- | --- | --- |
| 1 | `startup_empty.png` | 未ワークスペース状態 | "フォルダーを開く" ボタンや空の状態の UI 崩れ |
| 2 | `workspace_tree.png` | ネストしたフォルダを開いた状態 | インデントやファイルアイコンの適切な表示 |
| 3 | `editor_split_mode.png` | 長文 MD + Split Mode | ペイン分割の比率、スクロールバー、タブバーの崩れ |
| 4 | `diagram_rendering.png` | Mermaid / PlantUML | ラスタライズされたダイアグラム画像の解像度と比率 |

## 実装に関する注意事項

* CI でスナップショットの微小なアーキテクチャ差分（FreeType、GPU アンチエイリアス）により False-Negative が発生する可能性があるため、閾値調整を検討すること。
* macOS 固有ネイティブメニューアクション（`Option-Command-S`等）は、E2E レベルでは直接叩くのではなく対応する内部の `AppAction` 発出をエミュレートしてテストする。
