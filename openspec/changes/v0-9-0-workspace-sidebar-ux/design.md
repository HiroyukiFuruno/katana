## Context

現状の `render_workspace_panel` では、1 行目に `Workspace` / `ワークスペース` 見出しと折りたたみボタン、2 行目に履歴・更新・フィルター・検索・全展開・全閉を混在配置している。`search` と `recent workspace history` は既に機能として存在し、最近のワークスペース履歴は `settings.workspace.paths`、検索モーダルは `show_search_modal` により制御されている。

一方で、ワークスペースペインを閉じると専用の狭いトグル列だけが残り、検索や履歴の導線は消える。今回の要件では、添付イメージのような左側の縦レールに頻出導線を寄せ、ヘッダーから冗長なタイトルを除去したい。

## Goals / Non-Goals

**Goals:**

- ワークスペース表示切り替え・検索・履歴を左側の固定アクティビティレールへ再配置する
- ワークスペース見出し文字列を撤去し、ヘッダー操作の視認性を改善する
- 更新は左寄せ、全展開・全閉は右寄せ、フィルターは現状機能を維持したまま同一ヘッダー内で扱う
- 既存アイコン資産を流用し、左側レールでは一段大きいサイズで描画する

**Non-Goals:**

- ワークスペースツリーのノード表示仕様や検索アルゴリズムの変更
- 新しいアイコン資産の追加や SVG 自体の差し替え
- 最近のワークスペース履歴の保存形式変更
- フィルターの正規表現仕様や結果計算ロジックの変更

## Decisions

### 1. 左端に専用アクティビティレールを追加する

ワークスペースペインの有無にかかわらず使えるよう、既存の `workspace_collapsed` 代替として常時表示の細い左レールを追加する。ワークスペースペイン内部のヘッダーに機能を寄せる案もあるが、その場合はペインを閉じた瞬間に検索・履歴の導線が消え、今回の UX 改善意図に反する。

### 2. レール上のボタンは既存アイコンを一段大きく再利用する

ボタン種別は以下とする。

- ワークスペース表示切り替え: 既存の `ChevronLeft` / `ChevronRight`
- 検索: 既存の `Search`
- 履歴: 既存の `Document`

左レールでは `IconSize::Medium` を用い、既存ヘッダー側の密な小型ボタンより一段大きく描画する。新しいアイコンを導入する案もあるが、今回の要件では「現状のアイコンを維持」が優先されるため採用しない。

### 3. ワークスペースヘッダーは左右 2 グループへ再編する

ワークスペース見出し文字列を削除し、操作ヘッダーは次の構成に整理する。

- 先頭側: 更新、フィルター
- 末尾側: 全展開、全閉

検索と履歴は左レールへ移すため、ヘッダーの横幅を圧迫しない。フィルター入力欄は現行どおりボタン直下に出し、既存の `filter_enabled` / `filter_query` / `filter_cache` をそのまま利用する。

### 4. 履歴メニューは既存データと既存アクションを再利用する

最近のワークスペース履歴は `settings.workspace.paths` をそのまま表示し、`OpenWorkspace` / `RemoveWorkspace` を再利用する。新しい履歴ストアや別 UI 状態を持ち込まず、レール側のアンカー位置だけを変える。

### 5. レールの状態表現は「残ること」を優先する

ワークスペースペインを閉じても左レールは残し、同じボタンで再表示できるようにする。推奨挙動として、ワークスペース表示切り替えボタンにはアクティブ時の塗りやホバー説明を持たせ、現在の表示状態が判断しやすいようにする。

## Risks / Trade-offs

- **[Risk] 左レールの追加で横幅が減る** → レール幅は固定の最小限に抑え、見出し文言削除で相殺する。
- **[Risk] `ChevronLeft` / `ChevronRight` は VS Code 的な Explorer アイコンほど意味が直感的ではない** → 要件どおり現行アイコンを優先し、アクティブ塗りとツールチップで補う。
- **[Risk] ペイン内とレール上でボタン責務が分散し、導線が曖昧になる** → 「レール = 主要導線」「ヘッダー = 現ワークスペース固有操作」と責務を明確に分ける。
- **[Risk] ペインを閉じた状態での検索・履歴起動が未テストの分岐になる** → UI テストにレール起点の操作パスを追加する。

## Migration Plan

- 永続化データ形式の変更は行わないため、設定ファイルの移行は不要。
- 既存の検索ショートカットやワークスペース履歴更新順は維持し、UI 導線のみ段階的に差し替える。

## Open Questions

- 履歴が 0 件のとき、履歴ボタンを非活性表示で残すか、表示自体を省略するか。レイアウトの安定性を優先するなら、非活性表示で残す案が妥当。
