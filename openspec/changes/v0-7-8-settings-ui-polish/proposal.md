# プロポーザル: Settings UIのレイアウト調整と透過度制御・キャッシュ削除機能の復帰

## 背景・課題

v0.7.7の緊急リリース（画像キャッシュ競合回避等）を優先したため、バックエンドの内部ロジックとして追加された以下の機能がUI側に露出されていませんでした。
- UI透過度の全体オフセット調整機能（テーマコントラストの微調整）
- HTTPキャッシュディレクトリの完全削除（リセット）機能

また、ユーザーからのフィードバックにより、設定画面「カスタムテーマ編集」におけるカラーピッカーのラベル文字列とピッカー要素（四角形）の垂直アライメントがずれている（ラベルが上に片寄っている・上下中央揃えになっていない）レイアウト上の不具合が指摘されました。

## 今回の変更内容 (v0.7.8)

本改修では以下のUI統合とレイアウト調整を実施します。

1. **追加機能のGUI復帰（UI統合）**
   - **透過度補正スライダー**: 外観 (Appearance) セクションのテーマ設定下に `-100%` から `+100%` までの `ui_contrast_offset` を操作できるUIを追加し、テーマ全体の視認性・コントラストを微調整できるようにします。
   - **キャッシュ削除ボタン**: システム (System) セクション等に、蓄積されたHTTP画像キャッシュ（アバターやバッジ等）を安全に一括削除するボタンを追加します。
2. **Settings UI のレイアウト修正**
   - **垂直中央揃え (Vertical Alignment)**: `render_custom_color_editor` 等の `egui::Grid` において、ラベル文字列とカラーピッカー要素を垂直方向（上下）の中央で綺麗に揃えるよう、レイアウト指定（`egui::Align::Center`やラッパーAreaの導入）を修正します。

## 期待される効果

- 透過度の調整を取り戻すことで、各ユーザーのモニター環境やコントラスト感度に合わせた柔軟なテーマ表示が実現します。
- カスタムカラーパレットのUIが整然と揃うことによって設定画面の見た目の品質が高まります。
- 溜まったキャッシュをGUIからオンデマンドに解放・リフレッシュ可能になります。
