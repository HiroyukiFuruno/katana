## MODIFIED Requirements

### Requirement: タブのドラッグ並び替え
タブをドラッグ&ドロップして並び順を変更できる。左右双方向のドラッグが可能で、移動中はアニメーションフィードバックが提供される。

#### Scenario: タブの右方向ドラッグ移動
- **WHEN** ユーザーがタブをドラッグし、右側の位置にドロップする
- **THEN** タブがドロップ位置に移動し、他のタブが適切にシフトする

#### Scenario: タブの左方向ドラッグ移動
- **WHEN** ユーザーがタブをドラッグし、左側の位置にドロップする
- **THEN** タブがドロップ位置に移動し、他のタブが適切にシフトする

### Requirement: ピン留めタブ
タブをピン留めして固定位置に配置できる。ピン留めタブはアイコンとタイトルのコンパクト表示となり、閉じる系操作の対象外となる。ピン留めを解除しない限り、削除操作が保護される。

#### Scenario: タブのピン留め
- **WHEN** ユーザーがタブを右クリックし「ピン留め」を選択する
- **THEN** タブがタブバーの左端に移動し、コンパクト表示（アイコン＋タイトル）になり、閉じるボタン（×）が非表示になる

#### Scenario: ピン留め中のタイトル表示
- **WHEN** ユーザーがピン留めされたタブにホバーする
- **THEN** タイトルがツールチップとして表示される

#### Scenario: ピン留め解除
- **WHEN** ユーザーがピン留めされたタブを右クリックし「ピン留め解除」を選択する
- **THEN** タブが通常のサイズに戻り、閉じるボタン（×）が再表示される

#### Scenario: ピン留め中のタブ削除ガード
- **WHEN** ユーザーがピン留めされたタブを閉じようとする（ショートカットキー等）
- **THEN** 操作が無視される（閉じるボタンが非表示のため直接操作不可）

## ADDED Requirements

### Requirement: Source Treeディレクトリアイテムのコンテキストメニュー
Source Tree（ファイルツリー）上のファイルまたはディレクトリアイテムを右クリックすると、Workspaceペインと同等のコンテキストメニューが表示される。これはタブバー上のタブではなく、左ペインのファイルツリー上のアイテムが対象である。

#### Scenario: Source Treeファイルアイテムの右クリックメニュー表示
- **WHEN** ユーザーがSource Tree（左ペインのファイルツリー）内のファイルアイテムを右クリックする
- **THEN** Workspaceペインのファイルコンテキストメニューと同等の操作（開く、コピー、削除、名前変更等）が表示される

#### Scenario: Source Treeディレクトリアイテムの右クリックメニュー表示
- **WHEN** ユーザーがSource Tree内のディレクトリアイテムを右クリックする
- **THEN** Workspaceペインのディレクトリコンテキストメニューと同等の操作（新規ファイル作成、ディレクトリ削除、名前変更等）が表示される

#### Scenario: Source Treeコンテキストメニューからファイルを開く
- **WHEN** ユーザーがSource Tree内のファイルアイテムのコンテキストメニューから「開く」を選択する
- **THEN** 対象ファイルがエディタタブで開かれる（既に開いている場合はアクティブになる）
