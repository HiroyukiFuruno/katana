## Context

Katana v0.7.7ではテーマのハードコードを撤廃しましたが、デフォルトの透過率（RGBa）の視認性に関する課題が浮上しました。さらに、画像キャッシュ (`http_cache_loader`) の削除失敗エラー (`os error 66`) もバックグラウンドで発生しており、設定画面を通じた統合的なシステム補正・メンテナンス機能（UI透明度補正、およびキャッシュ制御）の需要が高まっています。

## Goals / Non-Goals

**Goals:**

- UIで展開される全てのRGBaアルファチャンネルに対し、設定画面から一律のオフセット補正（`-100%` 〜 `+100%`）を適用できるようにする。
- ユーザーが設定画面からボタン一つで HTTP 画像キャッシュディレクトリのクリアを実行できるようにする。
- キャッシュの有効期間を制御する設定を追加し、無限に増殖・古くなるキャッシュを制御する。

**Non-Goals:**

- 画像以外のローカルファイルキャッシュ（AST解析結果など）の包括的なガベージコレクション機構の構築（今回はHTTP画像キャッシュに絞る）。
- 不透明色（Rgb）に対する自動明度補正。

## Decisions

- **コントラスト補正ロジック**:
  - `ui_contrast_offset: i8` を追加。計算は `new_alpha = clamp(original_alpha + 255 * (offset / 100.0), 0, 255)` を採用する。
- **キャッシュ管理UIとロジック**:
  - `settings_window.rs` の「システム (System)」セクション配下に機能を追加する。
  - `clear_cache` 関数を `http_cache_loader` 等のモジュールに定義し、UIスレッドから非同期、または安全なタイミング（ファイルロック解放時）で `fs::remove_dir_all` 処理を呼び出す。
  - OS Error 66 (Directory not empty) 対策として、削除前にディレクトリ内の全ファイルハンドルが閉じられていることを確認するか、強制再帰削除の堅牢化（`walkdir`等を用いた確実なファイルunlink後のディレクトリ削除）を実施する。

## Risks / Trade-offs

- **Risk:** キャッシュ削除時のファイルロック競合 (OS Error 66等の再発)
  - **Mitigation:** 実装時に標準の `fs::remove_dir_all` だけでなく、中身のファイルを確実に個別 `remove_file` してからディレクトリを消す安全なリトライ機構、もしくはエラーを無視して消せるものだけ消すベストエフォート型のクリア方式を検討する。
