## Definition of Ready (DoR)

- **前提条件**: `desktop-viewer-polish-v0.1.0` の内容が `main` ブランチにマージされていること。
- **タスクの順序性**:
  - `タスク2 (ワークスペースローディング表示)` および `タスク3 (コンテキストメニュー)` は、`タスク1 (ワークスペース永続化)` によって複数のディレクトリが安定して管理できる状態になってから着手する。
  - `タスク4 (ダイアグラム最適化)` は独立して実装可能だが、ファイル操作が頻発するワークスペース関連実装後に並行して動作確認を行うこと。

## Branch Rule

タスクグループ（##単位）= 1セッションで以下のサイクルを回す:

1. **ベースブランチ作成**: 最初に `desktop-viewer-polish-v0.2.0` を `master` から作成する
2. **タスクブランチ作成**: 各タスクごとの作業は `1` のブランチから `desktop-viewer-polish-v0.2.0-task{N}` を派生させる
3. **実装 / PR / セルフレビュー / マージ**: `/opsx-apply` (マージ先は `master` ではなく `1` のブランチとする。マージ完了後は `1` のベースブランチへ切り替え、最新化を行う)
