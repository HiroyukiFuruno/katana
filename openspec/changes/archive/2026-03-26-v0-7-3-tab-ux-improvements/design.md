## Context

KatanAはegui製のデスクトップMarkdownエディタ。タブバーは独自実装で、Workspaceペイン（ファイルツリー）との連携やピン留め、右クリックコンテキストメニューをすでに部分的に持つ。

現状の課題：

1. 特定環境（主にmacOS特定GPUドライバ）でegui `TextEdit` / シンタックスハイライトの背景が描画されない
2. タブのドラッグ移動が右→左方向のみ対応。左→右移動が機能しない。移動中のアニメーション/UX品質も低い
3. Source Tree（ディレクトリツリー）上のタブアイテムにはコンテキストメニューが存在しない
4. タブグルーピング機能がない（Chromeライクなグループ概念）
5. セッション復元機能がない（前回終了時の状態を次回起動時に再現できない）
6. ピン留めタブを直接削除できてしまう・ピン留め中にタイトルが見えないUX問題

## Goals / Non-Goals

**Goals:**

- ハイライト背景非表示バグの根本原因特定と修正（環境依存の描画問題）
- タブの双方向移動とUI/UX品質の統一
- Source TreeタブへのWorkspace同等コンテキストメニューの追加
- タブグルーピング（色、グループ名、ワークスペース永続化）
- セッション復元（最後に開いていたタブ状態の永続化と復元）
- ピン留めUX改善（削除ガード、タイトル表示）

**Non-Goals:**

- タブの縦方向配置（現状の横タブバー構成を維持）
- グループ間のドラッグ&ドロップ（初版では手動割り当てのみ）
- クラウド同期（セッション/グループ情報はローカル永続化のみ）

## Decisions

### ハイライト背景バグ調査方針

egui の `Painter::rect_filled` が環境によって描画されない報告を起点に調査する。
代替案として `Frame::fill` を使った背景描画への切り替えも検討するが、まずログ・再現条件の絞り込みを優先する。

> **決定**: 再現環境でのスタックトレース取得と egui issue tracker 照合を先行。修正方針はバグ種別により変動を許容。

### タブ移動の双方向化

現状の実装は配列インデックスの swap のみ。左→右移動が機能しない原因は配列境界チェックのオフバイワンと推測。
移動時のアニメーションはegui の標準 `lerp` を活用し、ドラッグ中のズレを視覚フィードバックとして提供する。

> **決定**: 既存のドラッグロジックを修正し、双方向移動と移動中スナップアニメーションを統一実装。

### タブグルーピング

Chromeと同等の概念：

- グループは `TabGroup { id, name, color, tab_ids }` で表現
- 色は8プリセット（赤・橙・黄・緑・青・紫・ピンク・グレー）＋カスタムカラーパレット
- グループ情報はワークスペース設定ファイル（JSON/TOML）に永続化

> **決定**: カスタムカラーはフリー入力より egui の ColorPicker を使用。プリセット選択UIを優先、カスタムは折りたたみ展開。

### セッション復元

`SessionState { tabs: Vec<TabEntry>, active_tab: Option<TabId> }` を起動時に読み込み、終了時に書き込む。
既存の `AppState` の永続化層（設定ファイル）を拡張して対応。

> **決定**: `last_session.json` をアプリデータディレクトリに保存。ファイルが破損した場合は無視してデフォルト起動。

### ピン留め改善

- ピン留め中のタブに `×` ボタンを表示しない（または grayed-out にして操作不可にする）
- ピン留め中でもホバー時にタイトルをツールチップで表示
- ピン留め解除はコンテキストメニュー経由のみに統一

> **決定**: `×` ボタンを完全に非表示（UI簡潔化）、コンテキストメニューで「ピン留め解除」→ 解除後に `×` が出現する2ステップ保護。

## Risks / Trade-offs

- **[Risk] ハイライトバグの再現条件が限定的** → 複数GPU環境でのテストが困難。受け入れ条件をユーザー報告ベースとし、継続観察で対応。
- **[Risk] セッション復元がユーザーの意図しない状態を開く** → 設定でセッション復元のON/OFFを切り替え可能にする（初版はデフォルトON）。
- **[Risk] グループ永続化データの後方互換性** → スキーマにバージョンフィールドを追加し、旧形式無視による安全なフォールバックを実装。
