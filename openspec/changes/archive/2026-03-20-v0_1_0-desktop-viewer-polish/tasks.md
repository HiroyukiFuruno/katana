## Definition of Ready (DoR)

- **前提条件**: `main` ブランチにおいて、以前実装した基盤（`SettingsRepository`、`JsonFileRepository`等）が正しく動作していること。
- **タスクの順序性**:
  - `タスク3 (設定UIパネル)` は、`タスク1`、`タスク2`、および `タスク4 (レイアウト設定)` の内部ロジック（`AppSettings` への項目追加、UIへの動的反映メカニズム）が完了していることを前提に着手する。
  - `タスク5 (OSテーマ連動)` は、`タスク1 (テーマ設定)` が完了していれば任意のタイミングで着手可能。

## Branch Rule

タスクグループ（##単位）= 1セッションで以下のサイクルを回す:

1. **ブランチ作成**: `desktop-viewer-polish-v0.1.0-task{N}` を main から切る
2. **実装 / PR / セルフレビュー / マージ**: `/opsx-apply`

### 6.2 Apple Color Emoji フォントを読み込む

絵文字（🎉 😊 等）がレンダリングされない問題に対応する。
`cosmic_text` の `FontSystem` に Apple Color Emoji フォントを追加ロードすることで対応する。

- [x] 6.2.1 `katana-platform` の `os_fonts` モジュール（または新設の初期化処理）に Apple Color Emoji フォントのロード処理を追加する
- [x] 6.2.2 フォントファイルが存在しない環境（Linux/Windows 等）でも安全にスキップできるよう、存在チェックを行ったうえでロードする

**草案（`cosmic_text` 経由）:**

```rust
use cosmic_text::fontdb::Source;

font_system.db_mut().load_font_source(Source::File(
    "/System/Library/Fonts/Apple Color Emoji.ttc".into(),
));
```

**草案（`skia_safe` 経由）:**

```rust
use skia_safe::FontMgr;

let mgr = FontMgr::new();
let typeface = mgr.match_family_style("Apple Color Emoji", Default::default());
```

> **採用方針メモ**: `cosmic_text` はテキストレイアウトエンジンとして既に使用されているため、`Source::File` によるロードが最も自然。`skia_safe` 経由は描画バックエンドが Skia の場合に検討する。macOS 固有パスへの依存を避けるため、`APPLE_COLOR_EMOJI_PATH` 等の定数化も検討する。
