# OpenSpec Change Proposal: アプリ内リリースノート表示機能 (v0.8.0)

## Overview

KatanAアプリ内から直接最新のリリースノート（CHANGELOG）がエディタータブ上で読める新機能を追加します。専用のUIウィジェットを一から構築するのではなく、GitHubから取得したCHANGELOGをパースし、アプリのアコーディオン機能（`<details><summary>`記法）を用いて加工した**専用のMarkdownファイル（キャッシュ）を動的生成して、新規タブとして表示**します。

特に、アプリアップデート直後の再起動時には自動的にこのタブが生成・表示され、**未適用だったバージョン（前回起動バージョン 〜 最新バージョン）までの変更点のみが「開いた状態（open）」**となり、ユーザーがスムーズに変更内容を把握できるようにします。

## Motivation

現状、新しいバージョンがリリースされた際、ユーザーはGitHub上のリポジトリを開いてCHANGELOGを確認する必要があります。
これをアプリ内完結させ、かつKatanAが持つ強力なMarkdownプレビューの強み（タブ表示、テキスト検索、コピペ可能）を活かして、実装コストを抑えつつ最高のユーザー体験を提供します。

## Proposed Changes

- **CHANGELOGのフェッチとパース**:
  - `CHANGELOG.ja.md` (日本語環境) または `CHANGELOG.md` (それ以外) をGitHubから非同期フェッチ。
  - バージョンごとのセクション（`# [0.8.0]` 等）に分割してパースする。
- **アコーディオン付き専用Markdownの動的生成とキャッシュ化**:
  - 毎回パースするのは非効率なため、処理済みのMarkdownテキストを生成し、ローカルにキャッシュファイルとして保存する。
  - **開閉ロジックの制御**:
    - **アップデート再起動時**: 「前回起動したバージョン」から「最新バージョン」までの差分に該当するセクションには `<details open>` を付与し、それより過去の機能拡張は `<details>` (閉じた状態) にする。
    - **最新の状態で手動で開いた時**: 全てのバージョンを `<details>` (閉じた状態) にする。
- **エディターの新規タブへの統合**:
  - 生成されたキャッシュファイルを「読み取り専用バッファ」等としてKatanAの新しいタブで開く。
  - **タブ名の工夫**: Obsidianを参考にし、`📄 リリースノート 1.12.7`（英語なら `📄 Release Notes 1.12.7`）のように、アイコンとバージョンを含めた視認性の高いタブ名を設定する。
- **起動時のフックと各UIからの連携**:
  - **アップデート再起動時**: 自動的にリリースノートタブを前面に開く。
  - **更新確認（Update Check）画面**: アップデート通知ダイアログ内に「リリースノートを読む」導線（ボタン等）を追加し、クリックで対象バージョンのリリースノートタブをワンストップで開けるようにする。
  - **Helpメニュー**: `Help` 内に `Release Notes` アイテムを追加し、いつでもキャッシュを開いて全件の情報を閲覧可能にする。

## Dependencies / Open Questions

- インターネット接続時のHTTPリクエストクライアントへの依存。
- `previous_version` （アプリの前回起動バージョン）を保持・判定するための状態管理（設定ファイル等の更新ロジック）。
- オフライン時やフェッチ失敗時に、最後に成功したキャッシュを表示するか、エラータブを出すかのフォールバック設計。
