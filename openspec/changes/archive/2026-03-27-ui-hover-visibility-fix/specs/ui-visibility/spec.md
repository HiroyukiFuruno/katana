## ADDED Requirements

### Requirement: プレビュー画面での連携ホバー・アクティブ行のテーマ準拠背景描画
システムは、`egui_commonmark` を利用するプレビュー画面において、ハードコードされた固定色ではなく、ユーザーが選択・設定中のKatanAテーマ（`PreviewColors`）に完全に準拠したホバー背景色およびアクティブ行背景色を描画しなければならない（SHALL）。

#### Scenario: プレビュー画面での領域ホバー
- **WHEN** ユーザーがプレビュー画面上の特定の行（ブロック）にマウスカーソルを乗せた時
- **THEN** テーマで定義された `preview.hover_line_background` のRGBA色を用いた背景矩形がハイライトとしてブロック背面に描画される

#### Scenario: プレビュー画面でのアクティブ行追従（エディタ連動）
- **WHEN** ユーザーがエディタ側でキャレットを移動させ、連動してプレビュー側のアクティブ行が変化した時
- **THEN** テーマで定義された `preview.current_line_background` のRGBA色を用いた背景矩形がハイライトとしてブロック背面に描画される

## MODIFIED Requirements

### Requirement: エディタ画面の現在行・ホバー行視認性確保
システムは、ダークモードテーマ（`ThemeMode::Dark`）においては「暗色（黒）の透過色」ではなく「明色（白）の定率透過色（Alpha: 15〜25など）」を用い、暗い背景であっても現在行およびホバー行が自然に視認できるよう輝度のコントラストを確保しなければならない（SHALL）。
また、ライトテーマ（`ThemeMode::Light`）においても、同様に黒の透過度を適正量（Alpha: 15〜25など）へ調整し、視認性を担保しなければならない（SHALL）。

#### Scenario: ダークテーマ下でのエディタホバー
- **WHEN** ユーザーがダークテーマを選択し、テキストエディタの任意行をホバーした時
- **THEN** ホバーされた行の背景が、文字色やシンタックスハイライトを潰すことなく、明瞭でかつ眩しすぎない白ベースの透過色でハイライトされる
