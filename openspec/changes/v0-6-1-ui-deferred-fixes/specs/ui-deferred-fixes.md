# v0.6.1 UI Deferred Fixes

## 概要

Katana UI の表示改善のうち、即時性が低い（またはリファクタリングの影響範囲が大きすぎるために後回しにされた）P3以下のバグ修正や仕様変更タスクを格納する。

## 要件

以下の機能改善を実装し、各種テストでデグレードが起こらないことを確認する。

- **[DEFERRED: P3] インラインコードのベースライン表示位置の修正**
  - アラインメントの都合上、本バージョンでは安定性を重視して強制 `BOTTOM` 揃えを維持するが、本来は CJK/英数の混植時に文字が下ズレして見えることを防ぐため、`egui::Align::Center` が期待値である。
  - これは付随する単体テスト (`markdown_text_uses_bottom_vertical_alignment_...`) のアサーション修正（テストルールの見直し）を含んだ中規模の修正となるため、v0.6.1へ見送る。

- **[DEFERRED: P4/Feature] テーブル要素の完全修復（アラインメントと縦罫線）**
  - 現状の `egui_commonmark` はテーブル全体のレイアウト管理を `egui::Grid` に委譲しているため、各セルの幅に基づく動的なアラインメント（中央寄せ・右寄せ）を正確に制御できず、また縦のセパレータ（罫線）を表示する機能が存在しない。
  - 本課題を解決するには、テーブル描画エンジンを列の厳密管理や境界線描画に適した `egui_extras::TableBuilder` へと大幅に切り替える（リファクタリングする）必要があるため、単なる表示バグ修正の枠を超えると判断し、機能追加とみなして次回（v0.6.1段階）への見送りとした。
