# OpenSpec Change Proposal: テーマ設定の構造最適化およびバグフィックス (v0.7.7)

## Overview

カスタムテーマ機能を拡張・整理し、反映漏れバグの修正と設定UIのグルーピング化を行うリリースです。コードブロック等の背景色がカスタムテーマで反映されない不具合を解消するとともに、増大した色設定項目を `preview`, `code`, `system` メタファーでグループ化し、内部のJSONスキーマおよびUIの両面で整理します。さらに旧バージョンの設定からのマイグレーション機構も備えます。また、テーマ変更に伴い、プレビューの表示方式（分割表示時）のレイアウト改善も実施します。

## Motivation

v0.7.6にてカスタムテーマ機能が導入されましたが、「コードの背景色が変わらない」などの反映漏れや、フォントと背景色がフラットに並んでおり設定UIが煩雑になっている課題があります。またeguiのデフォルト値や内部固定値に依存している色も多く存在します。これらをカテゴリ分け（グルーピング）し、より直感的にテーマを「育てられる」基盤を整える必要があります。

## Proposed Changes

- **設定UIと内部スキーマのグループ化**:
  - 設定項目をカテゴリ（`preview` / `code` / `system` 等）に分類し、UI上でわかりやすく階層化（アコーディオン等）して表示。
  - カスタムテーマの内部JSON構造についても同様のグループ化された構造 (`PreviewColors`, `CodeColors`, `SystemColors` 等) へ再設計。
- **後方互換性とマイグレーション**:
  - 旧フラット構造のカスタムテーマJSON（v0.7.6形式）を読み込んだ際に、新しい階層型構造へ自動変換（migrate）する機構の実装。
- **固定値・デフォルト値のカスタマイズ化**:
  - 従来ソースコード内でハードコードされていた色や、egui側のデフォルトテーマに依存していた色を洗い出し、設定可能な項目として追加。
- **プレビュー分割表示の改善**:
  - テーマの不具合修正と連動し、分割表示時のプレビューレイアウトが最適になるようUIを微調整。

## Dependencies / Open Questions

- `serde` を用いたJSON構造の移行（`#[serde(default)]` やカスタムデシリアライザの活用）実装が必要です。
