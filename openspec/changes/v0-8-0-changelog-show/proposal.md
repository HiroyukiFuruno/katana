# OpenSpec Change Proposal: アプリ内リリースノート表示機能 (v0.8.0)

## Overview

KatanAアプリ内から直接リリースノート（CHANGELOG）を参照できる新機能を追加します。ヘルプメニューから「最新のリリースノート（Latest）」および「過去のリリースノート一覧（All）」にアクセス可能にし、OSの言語環境に応じて日本語または英語のCHANGELOGをGitHubから動的にフェッチ・レンダリングして表示します。さらに、この表示コンポーネントを再利用し、更新確認（Update Check）画面からも現在のバージョンと新しいバージョン間の「差分リリースノート」を確認できる仕組みを提供します。

## Motivation

現状、新しいバージョンがリリースされた際、ユーザーはGitHub上のリポジトリを開いてCHANGELOGを確認する必要があります。アプリ内完結でスムーズに更新内容を把握できるようにすることで、ユーザー体験の向上とアップデートの心理的ハードルを下げることを主眼としています。

## Proposed Changes

- **メニュー追加**:
  - `Help` メニューに `Release Notes` アイテムを追加し、サブメニューとして `Latest` と `All` の導線を設ける。
- **オンラインCHANGELOGのフェッチ・パース**:
  - GitHubの生ファイルへのHTTPリクエスト（言語設定が日本語なら `CHANGELOG_ja.md`、それ以外なら `CHANGELOG.md`）を行い内容を取得。
  - マークダウン文字列からバージョン単位のヘッダー（例 `# [0.7.0]`）を境界域としてセクションを分割・構築。
- **専用UIコンポーネントの実装**:
  - 一覧画面では、バージョンごとにアコーディオン表示（常に最新の1件目のみ開いた状態）でレンダリングする共通UIを構築する。
  - スコープオプション（全件、最新N件、バージョンAからBまでの差分、等）をオプション引数として持ち、ロジックを再利用可能に設計する。
- **更新チェック画面への統合**:
  - アップデートダイアログ内に「リリースノートの差分を表示」するためのボタンまたはインライン表示領域を追加し、上記共通UIコンポーネントを呼び出して差分のみを描画する。

## Dependencies / Open Questions

- インターネット接続時のHTTPリクエストクライアント（`reqwest` などの既存実装）への依存。
- アプリ単体起動時のフェッチ遅延におけるプログレス表示UIや、オフライン時の適切なエラーハンドリングの考慮。
