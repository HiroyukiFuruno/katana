## Context

本リポジトリはOSS（オープンソースソフトウェア）化を予定しており、グローバルな開発者に幅広く利用・貢献してもらえるようにするための整備が必要です。そのため、リポジトリの主要なドキュメントおよびソースコード上のコメントの第一言語を英語（English First）へと統一します。  
現在、`docs` ディレクトリにはリポジトリの使用方法とは異なる「今後の計画」や「仕様などの思考プロセス」が含まれており、OSSとしての構成を混乱させる原因となっています。これらは本来 `openspec` 上で管理されるべきであり、`docs` ディレクトリは「使い方や機能の説明のみ」のクリーンな状態にする必要があります。
また、メインの運用者として日本人ユーザーを優遇・サポートすべく、例外として `docs` 配下および `README.md` にのみ、日本語版ドキュメントの並列管理ルールを設けることとします。

## Goals / Non-Goals

**Goals:**
- `docs/` ディレクトリ配下にある、現在ではなく将来を対象とした設計書や仕様、ロードマップなどの不要なドキュメントをパージし、すべて `openspec` に移行する。
- リポジトリにおける一般ドキュメント（`README.md` 等）やソースコード上の全コメントを英語表記へと変更する。
- `docs/` ディレクトリと `README.md` についてのみ、日本語を並行管理する体制（`*.ja.md` など）を整える。
- `openspec` 自体やその他の `.gitignore` で指定されたファイル、`i18n` ディレクトリなどの設定ファイルは英語強制化や翻訳の**対象外**とし、手付かずのままとする。

**Non-Goals:**
- 各ソースコードの自動一括翻訳ツールを組み込み、その結果を無条件にコミットすること（レビューできない大規模な変更のブロック）。
- ソースコード上のプログラムロジックそのものや変数名の変更（あくまでコメントのみが対象）。

## Decisions

- **並列ドキュメントの命名規則**: `docs/` 内部およびルートディレクトリの日本語ドキュメントには、必ず `*.ja.md` サフィックスを付けることとします（例： `README.md` と `README.ja.md`）。ディレクトリを分割する（例： `en/` `ja/`）ことよりも、同一階層でファイル名を統一する手法を選択することで、同一機能の言語の対応関係を一目で確認できるようにします。
- **対象外範囲の厳格な設定の適用**: 翻訳およびスコープ対象の検索処理（`grep`等）において、`.gitignore` が管理するディレクトリ・ファイル（`openspec/`等）と `i18n/` は**完全に除外**します。この `openspec` 管理自体も除外の例外であるため、ドキュメント生成言語は作成者・投稿者の母国語である日本語を許可します。

## Risks / Trade-offs

- **Risk: 非同期なドキュメント状態への陥り**: 2言語展開にすることで、一方が最新になり他方が遅れる可能性があります。 Mitigation: `docs` または `README.md` への変更に関するプルリクエストでは、必ず英語版と日本語版（`*.ja.md`）の両方に変更を加えることを必須ルールとして定義します。
- **Risk: 自動翻訳によるソースコードの脈絡崩壊**: ソースコメントの一括翻訳は技術用語の誤訳や脈絡破綻を引き起こす恐れがあります。 Mitigation: 一度の巨大なコミットではなく、クレート・ディレクトリ単位で検索をかけ、翻訳内容の意味を人力またはツールで内容確認しながら段階的に翻訳（Phased Approach）してコミットとレビューを行います。
